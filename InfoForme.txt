2️⃣ À quoi sert Maestro

Maestro n’est pas nécessaire pour faire tourner ton app sur un émulateur. C’est un outil de tests automatisés pour mobile.
Concrètement :

Il simule des interactions sur ton app : cliquer sur des boutons, remplir des formulaires, faire défiler des écrans…

Il vérifie que ton app réagit correctement et produit des rapports de tests.

Les fichiers .bat ou .ps1 que tu as lancés sont des scripts pour exécuter ces tests avec Maestro.

Donc si le script a fonctionné, c’est probablement parce que :

Il est écrit pour détecter si Maestro est installé et sinon il fait quelque chose de “minimal” ou affiche un message, ou

Le script exécute simplement des vérifications locales sans Maestro.
2️⃣ Ce que ton .bat est censé faire

Typiquement, ce type de script :

Vérifie que Maestro est installé sur ton ordinateur.

Lance Maestro pour exécuter les tests définis dans le projet (souvent dans un dossier .maestro ou tests).

Génère un rapport des tests, par exemple un fichier result.xml.

En regardant ton dossier, tu as déjà result.xml. Ça veut dire que le script a probablement exécuté quelque chose, même si Maestro n’est pas complètement installé, ou qu’il a utilisé un mode “mock” ou interne.

C:\Program Files\Eclipse Adoptium\jdk-25.0.0.36-hotspot

$env:PATH="$env:JAVA_HOME\bin;$env:PATH"
